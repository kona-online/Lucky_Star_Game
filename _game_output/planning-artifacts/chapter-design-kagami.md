# 카가미 챕터 세부 기획 — 스테이지형 QTE 츳코미

- **작성일**: 2026-07-01
- **상태**: 확정
- **MinigameBase 구현 대상**: `scenes/chapters/kagami/kagami.gd`

---

## 1. 챕터 개요

| 항목 | 내용 |
|---|---|
| 장르 | 스테이지형 QTE (Quick Time Event) |
| 배경 설정 | 히이라기 카가미, 교실/방과후에서 친구들의 엉뚱한 발언을 듣다가 참지 못하고 츳코미를 날린다 |
| 핵심 목표 | 3스테이지를 모두 클리어 — 각 스테이지에서 친구의 "헛소리" 대사가 끝나는 타이밍에 맞춰 츳코미 키를 입력한다 |
| 플레이 시간 | 약 2~3분 (스테이지 3개 × 40~50초 + 연출 — 재시도 없는 정상 진행 기준) |
| 실패 시 재시도 | 스테이지 단위 재시도 가능 (실패한 스테이지부터 재시작). 재시도 횟수 제한 없음 — 의도된 설계. |

---

## 2. 전체 구조

챕터는 3개의 독립적인 스테이지로 구성된다. 스테이지마다 헛소리를 치는 캐릭터와 대사가 다르다.

```
[챕터 진입]
    │
    ▼
[스테이지 1] ──(클리어)──► [스테이지 2] ──(클리어)──► [스테이지 3] ──(클리어)──► [챕터 클리어]
    │                          │                          │
 (실패)                     (실패)                     (실패)
    │                          │                          │
 [재시도/포기]              [재시도/포기]              [재시도/포기]
```

각 스테이지 구성:
1. **빌드업**: 친구 캐릭터의 대사/행동이 시작된다 (카가미는 경청 중)
2. **타이밍 윈도우**: 헛소리가 절정에 달하는 순간, 입력 판정 구간이 열린다
3. **판정**: 정확도에 따라 결과 분기

---

## 3. 핵심 상태 변수

| 변수 | 타입 | 설명 |
|---|---|---|
| `stage_num` | int 1~3 | 현재 스테이지 번호 |
| `stage_phase` | enum | `BUILDUP` / `WINDOW_OPEN` / `JUDGING` / `RESULT` |
| `input_time` | float | 타이밍 윈도우 진입 후 플레이어가 키를 누른 시각 (초). 미입력 시 -1. |
| `window_duration` | float | 판정 윈도우 지속 시간. 스테이지별로 다름. |

---

## 4. 판정 시스템

### 타이밍 윈도우

각 스테이지의 헛소리 절정 시점을 기준으로 판정 윈도우가 열린다.

```
[빌드업 대사 진행] ──► [윈도우 열림] ──► [윈도우 닫힘]
                              ↑
                        이 구간 안에 키 입력 필요
```

윈도우는 헛소리의 "마지막 한마디"가 끝나는 시점 직후 열리도록 설계 — 말이 끝나자마자 치는 즉각적인 츳코미 느낌.

### 입력 정확도 판정

윈도우 진입 시각 기준 `input_time`의 위치에 따라 3단계로 나뉜다.

| 판정 | 조건 | 결과 |
|---|---|---|
| **PERFECT** | 윈도우 앞 1/3 구간 내 입력 | 스테이지 클리어 + 풀 연출 |
| **GOOD** | 윈도우 나머지 2/3 구간 내 입력 (1/3 이후 ~ 윈도우 종료 전) | 스테이지 클리어 + 약한 연출 |
| **MISS** | 윈도우가 닫힐 때까지 입력 없음 | 실패 |

> **EARLY/LATE 입력 처리**: 윈도우가 열리기 전(빌드업 중) 또는 윈도우가 닫힌 후의 키 입력은 모두 **무시**한다 — 패널티 없음. MISS는 "윈도우 내 미입력" 한 가지 경우만 해당한다.

### 윈도우 지속 시간 (난이도)

스테이지가 진행될수록 윈도우가 짧아진다.

| 스테이지 | 윈도우 지속 시간 |
|---|---|
| 1 | 1.2초 |
| 2 | 0.9초 |
| 3 | 0.6초 |

## 5. 스테이지별 구성

### 스테이지 1 — 코나타의 게임 근황 토크

| 항목 | 내용 |
|---|---|
| 헛소리 주체 | 이즈미 코나타 |
| 빌드업 내용 | 코나타가 어젯밤 새벽 4시까지 게임했다고 자랑하며 수업 중 졸린 이유를 당당하게 설명한다 |
| 절정 대사 (예시) | "그러니까 졸린 거잖아~ 당연하지 않아?" |
| 카가미 츳코미 | "당연하지 않거든!!!" |
| 윈도우 | 1.2초 |
| 연출 | PERFECT 시 카가미가 머리를 쥐어뜯는 과장된 리액션 |

### 스테이지 2 — 츠카사의 엉뚱한 착각

| 항목 | 내용 |
|---|---|
| 헛소리 주체 | 히이라기 츠카사 (카가미의 동생) |
| 빌드업 내용 | 츠카사가 시험 공부를 열심히 했는데 교과서를 완전히 잘못 봤다는 사실을 해맑게 말한다 |
| 절정 대사 (예시) | "그래서 역사 시험에 수학 공식을 잔뜩 써왔어~!" |
| 카가미 츳코미 | "어떻게 그걸 모르고 시험장을 들어가!!!" |
| 윈도우 | 0.9초 |
| 연출 | PERFECT 시 카가미가 이마를 짚는 리액션 |

### 스테이지 3 — 미유키의 뜬금없는 박식함

| 항목 | 내용 |
|---|---|
| 헛소리 주체 | 다케아라 미유키 |
| 빌드업 내용 | 자연스러운 대화 중 미유키가 전혀 관계없는 심화 지식을 매우 진지하게 설명하기 시작한다 |
| 절정 대사 (예시) | "…그래서 문어의 혈액에는 헤모시아닌이 함유되어 있어서 파란색이랍니다, 후후." |
| 카가미 츳코미 | "갑자기 왜 문어 얘기야!!!" |
| 윈도우 | 0.6초 |
| 연출 | PERFECT 시 카가미가 손가락으로 가리키며 소리치는 리액션 |

> **스테이지 대사/내용은 초안**. 실제 애니메이션 에피소드 참고해 조정 가능. 핵심은 "말이 끝나는 타이밍에 츳코미를 날리는" 리듬감.

## 6. 화면 구성

**시점**: 카가미와 친구들이 좌우로 마주보는 2D 대화 구도. 카가미는 화면 오른쪽, 헛소리 주체는 화면 왼쪽.

```
┌─────────────────────────────────────────────────────┐
│  [스테이지 1/3]                      [남은 시간 없음]  │ ← 진행 표시 (좌상단)
│                                                     │
│  [코나타/츠카사/미유키]      [카가미]                  │
│   (좌측, 대사 말풍선)       (우측, 반응 대기 표정)      │
│                                                     │
│       "그러니까 졸린 거잖아~                           │
│        당연하지 않아?"                                │
│                                                     │
│                                                     │
│              [▶▶▶ 타이밍 바 ◀◀◀]                    │ ← 윈도우 열릴 때만 표시
│                                                     │
│                    [SPACE — 츳코미!]                 │ ← 입력 힌트 (하단)
└─────────────────────────────────────────────────────┘
```

**타이밍 바**: 윈도우가 열리는 순간 화면 하단 중앙에 등장 (빌드업 중에는 표시되지 않음 — 조기 입력 시 시각 피드백 없음, 의도된 동작). 마커가 왼쪽(윈도우 시작)에서 오른쪽(윈도우 종료)으로 이동하며 경과 시간을 나타낸다. 왼쪽 1/3이 PERFECT 구간(노란색), 나머지 2/3이 GOOD 구간(녹색). 마커가 바 끝에 도달하면 윈도우가 닫히고 MISS 처리.

**판정 결과 연출**:
- PERFECT: 카가미 리액션 풀 컷 + 큰 "PERFECT!" 텍스트 플래시
- GOOD: 카가미 작은 리액션 + "GOOD" 텍스트
- MISS: 카가미가 멍하게 있다가 뒤늦게 츳코미 → "너무 늦었어..." 텍스트

## 7. 클리어 / 실패 조건 요약

| 조건 | 결과 |
|---|---|
| 스테이지 3 클리어 (PERFECT 또는 GOOD) | **챕터 CLEAR** |
| 윈도우가 닫힐 때까지 입력 없음 | 스테이지 FAIL |

**스테이지 실패 시 흐름**:
1. 실패 연출 (카가미 "아… 타이밍 놓쳤잖아" 혼잣말 컷)
2. "다시 도전" / "앨범으로 돌아가기" 선택지
3. "다시 도전" → 실패한 스테이지부터 재시작 (`stage_num` 유지, `stage_phase` = `BUILDUP` 초기화)
4. "앨범으로 돌아가기" → `on_fail()` → `get_tree().change_scene_to_file("res://scenes/album/album.tscn")` 직접 호출 (`return_to_album()` **호출 금지** — 클리어 기록 오기록 방지)

## 8. MinigameBase 인터페이스 구현

```gdscript
# kagami.gd (MinigameBase 상속)

func start():
    stage_num = 1
    _start_stage(stage_num)

func _start_stage(n: int):
    _is_running = true          # retry 경로에서도 반드시 복구
    stage_phase = StagePhase.BUILDUP
    input_time = -1.0
    window_duration = _get_window_duration(n)  # 1.2 / 0.9 / 0.6
    _play_buildup_sequence(n)  # 빌드업 대사 재생, 완료 시 _open_window() 호출

func _open_window():
    stage_phase = StagePhase.WINDOW_OPEN
    _show_timing_bar()
    await get_tree().create_timer(window_duration).timeout
    # 타이머 만료 시점에 이미 플레이어가 입력했으면 JUDGING 상태 — 이중 호출 방지
    if stage_phase == StagePhase.WINDOW_OPEN:
        _judge()  # 미입력(MISS) 경로에서만 도달

func _on_player_input():
    if stage_phase != StagePhase.WINDOW_OPEN: return  # 윈도우 외 입력 무시 (EARLY/LATE 모두)
    input_time = _elapsed_since_window_open()
    stage_phase = StagePhase.JUDGING  # 타이머 콜백의 이중 호출 차단
    _judge()

func _judge():
    var verdict = _calculate_verdict()  # PERFECT / GOOD / MISS
    if verdict == "MISS":
        _play_fail_cutscene()   # "아… 타이밍 놓쳤잖아" 연출 먼저
        on_fail()               # 연출 완료 후 UI 선택 분기
    else:
        stage_phase = StagePhase.RESULT  # 컷신 중 추가 입력 차단
        _play_result_cutscene(verdict)
        if stage_num == 3:
            on_clear()
        else:
            stage_num += 1
            _start_stage(stage_num)

func on_clear():
    _is_running = false
    return_to_album()

func on_fail():
    _is_running = false
    # _play_fail_cutscene() 완료 후 이 함수에 도달
    # UI 선택:
    #   "다시 도전" → _start_stage(stage_num) 호출 (stage_num 유지, _is_running 내부에서 복구)
    #   "앨범으로" → get_tree().change_scene_to_file("res://scenes/album/album.tscn") 직접 호출
    #   !! 절대 return_to_album() 호출 금지

func return_to_album():
    # on_clear() 경로에서만 호출.
    ProgressManager.mark_cleared("kagami")
    get_tree().change_scene_to_file("res://scenes/album/album.tscn")
```

## 9. 미결 사항 / 다음 라운드 이관

- **스테이지 대사 확정**: 위 대사는 컨셉 초안. 실제 애니메이션 에피소드의 대사 톤/분위기 참고해 작성 필요.
- **빌드업 구현 방식**: 대사를 Dialogic으로 처리할지, 타이머+Label 시퀀스로 처리할지 — 코나타 챕터와 동일하게 첫 프로토타입 시 결정.
- **타이밍 바 UI**: 마커 이동 속도, PERFECT/GOOD 구간 비율 — 수치 밸런싱은 playtesting 후 조정.
- **PERFECT 전용 보너스 여부**: 현재 PERFECT/GOOD 모두 클리어로 처리. 향후 PERFECT 개수를 엔딩 연출에 반영할지 검토 가능 — 스코프 초과 우려로 보류.
- **배경 에셋 범위**: 각 스테이지 배경(교실 or 복도 등), 코나타/츠카사/미유키 대사 중 스프라이트, 카가미 반응 스프라이트(대기/PERFECT/GOOD/MISS 4종).
